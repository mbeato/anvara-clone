       - `parseGender(raw: string): { male: number; female: number }` — parses "62% male, 38% female" format. Regex: `/(\d+)%\s*male/` and `/(\d+)%\s*female/`. Defaults to 50/50 if parse fails.
       - `parseAgeRange(raw: string): { min: number; max: number }` — parses "18–34" or "18-34" format. Split on en-dash or hyphen.
       - `INCOME_SCALE: Record<string, { level: number; label: string }>` — maps income strings to display values:
         - "mass-market" → level 1, label "Mass Market"
         - "premium" → level 2, label "Premium"
         - "affluent" → level 3, label "Affluent"
         - "ultra-high-net-worth" → level 4, label "Ultra HNW"

    1. Create `app/actions/offer.ts` — Server Action ("use server"):
       - Export `async function createOffer(propertyId: string, amount: number, terms: string)`
       - Import prisma from `@/lib/prisma`
       - Create a Thread first (required by FK constraint on Offer):
         ```typescript
         const thread = await prisma.thread.create({
           data: {
             subject: `Offer — ${new Date().toLocaleDateString()}`,
             propertyId,
             messages: {
               create: {
                 sender: "advertiser",
                 isAI: false,
                 content: `Initial offer: $${amount.toLocaleString()}. Terms: ${terms}`,
               },
             },
           },
         })
         ```
       - Then create the Offer:
         ```typescript
         const offer = await prisma.offer.create({
           data: {
             threadId: thread.id,
             amount,
             terms,
             status: "pending",
           },
         })
         ```
       - Return `{ success: true, offerId: offer.id }`
       - Wrap in try/catch — return `{ success: false, error: "Failed to create offer" }` on error
