```typescript
import "dotenv/config";
import { PrismaClient } from "./generated/client.js";
import { PrismaNeon } from "@prisma/adapter-neon";

const adapter = new PrismaNeon({
  connectionString: process.env.DIRECT_URL!,
});
const prisma = new PrismaClient({ adapter });

async function main() {
  // 1. Clear existing data (idempotent — delete in reverse FK order)
  await prisma.message.deleteMany();
  await prisma.offer.deleteMany();
  await prisma.thread.deleteMany();
  await prisma.package.deleteMany();
  await prisma.property.deleteMany();

  // 2. Create properties with nested packages
  // 3. Create threads with nested messages
  // 4. Create offers on threads
}

main()
  .then(async () => {
    await prisma.$disconnect();
    console.log("Seed complete");
  })
  .catch(async (e) => {
    console.error(e);
    await prisma.$disconnect();
    process.exit(1);
  });
```

IMPORTANT: Use `DIRECT_URL` (not `DATABASE_URL`) in the seed script — seed runs as a CLI operation and needs the direct (non-pooled) connection.

**Property data requirements (10-14 properties):**

Cover these categories with realistic Anvara-inspired listings:
- **Sports** (3-4): e.g., LAFC, NY Jets, Miami Open, Austin FC
- **Music/Entertainment** (2-3): e.g., Coachella, Rolling Loud, Jazz at Lincoln Center
- **Arts/Culture** (2): e.g., Art Basel Miami, Tribeca Film Festival
- **Food & Beverage** (2): e.g., NYC Wine & Food Festival, Taste of Chicago
- **Action Sports/Lifestyle** (1-2): e.g., X Games, Red Bull Rampage

For each property include:
- Realistic `name`, `tagline`, `description` (2-3 sentences)
- `category` from: "sports", "music", "arts", "food", "lifestyle"
- `region` from: "west", "east", "south", "midwest", "international"
- `location` with city/state
- `imageUrl` using Picsum with unique seeds: `https://picsum.photos/seed/{slug}/800/600`
- `audienceTotalReach`: realistic numbers (50K to 5M)
- `audienceAgeRange`: e.g., "18-34", "25-54", "21-44"
- `audienceGender`: e.g., "60% male, 40% female", "mixed", "55% female, 45% male"
- `audienceIncome`: e.g., "premium", "mass-market", "affluent"
- `priceFrom`: ranges from 5000 to 250000 (NEVER $0)
- `availability`: e.g., "Available Q3 2026", "Sold Out — Waitlist", "Limited Availability"
- `featured`: mark 3-4 as featured
- `tags`: 4-6 relevant tags per property

**Package data (2-3 per property):**
Each property gets 2-3 tier packages (e.g., "Title Sponsor", "Gold", "Silver" or "Presenting", "Official", "Community"):
- `name`: tier name
- `priceUsd`: must be >= property's `priceFrom`. Highest tier 3-5x the lowest.
- `inclusions`: 3-6 specific deliverables per tier (realistic: signage, social media, VIP access, broadcast mentions, etc.)
- `maxSponsors`: 1 for title/presenting, 3-5 for mid-tier, 8-15 for community/silver

**Thread data (3-5 threads):**
Create pre-seeded conversations connected to specific properties:
- Thread 1: LAFC — advertiser interested in Gold package (2-3 messages back and forth)
- Thread 2: Coachella or music event — negotiating custom activation (3-4 messages)
- Thread 3: Art Basel — initial inquiry about presenting sponsorship (2 messages)
- Thread 4 (optional): Food festival — discussing community package terms (2-3 messages)
- Thread 5 (optional): X Games — advertiser sending an offer (2 messages + 1 offer)

Each message:
- `sender`: "advertiser" or "property"
- `isAI`: true for property-side messages (they simulate AI responses)
- `content`: realistic business conversation about sponsorship terms, activations, pricing

**Offer data (1-2 offers):**
- At least 1 offer on a thread with realistic amount and terms
- `status`: "pending" (one in active negotiation)
- `terms`: e.g., "6-month activation period, logo on all digital materials, 2 VIP passes per event"

**Tone:** Professional but approachable — the data should feel like real marketplace conversations between brands and properties.
