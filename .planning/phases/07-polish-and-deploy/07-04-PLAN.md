<task type="auto">
  <name>Task 2: Verify deployed site</name>
  <files></files>
  <action>
After the user provides the deployed URL, verify the deployment by checking:

1. **Landing page loads:** `curl -s -o /dev/null -w "%{http_code}" https://[DEPLOYED_URL]` returns 200
2. **noindex meta tag present:** `curl -s https://[DEPLOYED_URL] | grep -i "noindex"` finds the robots meta tag
3. **Browse page loads:** `curl -s -o /dev/null -w "%{http_code}" https://[DEPLOYED_URL]/listings` returns 200
4. **Property detail loads:** `curl -s -o /dev/null -w "%{http_code}" https://[DEPLOYED_URL]/listings/coachella-music-festival` returns 200 (or any valid slug)
5. **AI chat page loads:** `curl -s -o /dev/null -w "%{http_code}" https://[DEPLOYED_URL]/ai` returns 200
6. **AI actually responds (OpenAI key works):** Send a POST to the AI chat endpoint to confirm OpenAI is functional, not just page load:
   ```bash
   curl -s -w "\n%{http_code}" -X POST https://[DEPLOYED_URL]/api/ai-chat \
     -H "Content-Type: application/json" \
     -d '{"messages":[{"role":"user","content":"What sponsorship properties do you have?"}]}'
   ```
   This must return HTTP 200 with a JSON body containing a `"text"` field (the AI response) and a `"recommendedSlugs"` array. If it returns 500, the `OPENAI_API_KEY` env var is missing or invalid.

If any page returns 500, check Vercel deployment logs for errors. Common issues:
- Missing `DATABASE_URL` → database pages fail
- Missing `OPENAI_API_KEY` → AI chat POST returns 500
- Missing `postinstall` → all pages with Prisma fail

Report results to user.
  </action>
  <verify>All six curl checks return 200. noindex meta tag found in HTML. POST to /api/ai-chat returns JSON with "text" field proving OpenAI is connected.</verify>
  <done>Deployed site is live with all pages loading correctly, noindex confirmed, and AI chat verified to produce actual OpenAI responses.</done>
</task>
