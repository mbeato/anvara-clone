# Phase 7.1 Plan 01: Listing Page Mobile Polish Summary

**Full mobile-first rewrite of the listings/browse page — replaced Embla with native scroll on mobile, compacted all cards/filters/text, fixed viewport overflow, moved listing count to filter bar**

## Performance

- **Duration:** ~45 min (iterative with user testing)
- **Started:** 2026-02-23T21:04:41Z
- **Completed:** 2026-02-23
- **Tasks:** 2 (plan) + extensive iterative rework with user
- **Files modified:** 9

## Accomplishments

- Root `<main>` element in layout.tsx gets `overflow-x-hidden min-w-0` — fixes pinch-zoom-out bug where content wider than viewport expanded the page
- Category carousel completely split: mobile uses native horizontal scroll of `w-28 aspect-[16/9]` image chips; desktop keeps Embla autoplay carousel
- Property cards compacted: `aspect-[16/10]` images, `p-1.5`, `text-[11px]` name, `text-[10px]` tagline, smaller badges, date hidden on mobile
- Property card skeleton updated to match new card dimensions
- Filter bar compacted: `h-8` triggers, `text-xs`, inline price slider with label + slider + range on one line
- Listing count moved from bottom of page to right of price slider in filter bar
- Category tab row: `min-w-0`, `px-0.5` scroll padding, `shrink-0` buttons
- Recommendations/listing strips: `w-[148px]` cards on mobile, `px-0.5` scroll padding
- Browse client: `px-2.5` container, `grid-cols-1 min-[420px]:grid-cols-2` breakpoint, centered pagination
- Loading skeleton updated to match all new mobile dimensions and grid breakpoints

## Files Modified

- `app/(app)/layout.tsx` — overflow-x-hidden min-w-0 on main
- `app/(app)/listings/loading.tsx` — matching mobile skeleton dimensions
- `app/(app)/listings/_components/browse-client.tsx` — container padding, grid breakpoints, listing count removed from bottom, pagination centered
- `app/(app)/listings/_components/property-card.tsx` — compact prices, smaller text/padding, hidden date on mobile
- `app/(app)/listings/_components/property-card-skeleton.tsx` — matching card dimensions
- `app/(app)/listings/_components/category-carousel.tsx` — native scroll on mobile, Embla on desktop
- `app/(app)/listings/_components/category-tab-row.tsx` — min-w-0, scroll padding, shrink-0
- `app/(app)/listings/_components/filter-bar.tsx` — compact mobile layout, listing count prop
- `app/(app)/listings/_components/recommendations-strip.tsx` — smaller mobile cards, scroll padding

## Decisions Made

- Embla carousel has a hard-coded `overflow-hidden` wrapper that clips items at every `basis-*` value on mobile. After 4 failed attempts (basis-[55%], basis-1/3, basis-full), abandoned Embla on mobile entirely in favor of native horizontal scroll
- Root main element needs `overflow-x-hidden` — without it, any child wider than viewport (e.g., Embla's internal wrapper) expands the page and enables pinch-zoom-out
- `min-[420px]` used as a custom breakpoint for 2-column grid — Tailwind's `sm` (640px) is too wide; most phones switch to 2-col at ~400px
- Listing count relocated from bottom pagination area to filter bar for better visibility

## Deviations from Plan

The original plan targeted only 2 files (loading.tsx, property-card.tsx). Actual work required a full mobile-first rewrite of 9 files after user testing revealed the entire page needed mobile treatment, not just the two originally scoped overflow issues.

## Issues Encountered

- Embla carousel's `overflow-hidden` wrapper is not configurable — it's baked into the CarouselContent component. This made it impossible to show partial items on mobile without clipping.
- Multiple rounds of incremental fixes made things worse ("going backwards") — the breakthrough was doing a complete mobile-first rewrite in one pass rather than patching desktop-first components.
