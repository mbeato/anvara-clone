Fix the price row in PropertyCard that can overflow at 375px when "From $100,000" + "Apr 2025" exceeds available width:

1. **Switch to compact price notation**: Change the priceFormatter from standard to compact notation (already proven in filter-bar.tsx):
   ```typescript
   const priceFormatter = new Intl.NumberFormat("en-US", {
     style: "currency",
     currency: "USD",
     maximumFractionDigits: 0,
     notation: "compact",
   });
   ```
   This turns "From $100,000" into "From $100K" and "From $25,000" into "From $25K" — significantly shorter.

2. **Add truncation safety net**: Update the price row flex container to handle edge cases:
   ```tsx
   <div className="flex items-center justify-between pt-1">
     <span className="text-xs font-medium min-w-0 truncate">{priceDisplay}</span>
     <span className="text-xs text-muted-foreground shrink-0 pl-1">{startDate}</span>
   </div>
   ```
   - `min-w-0 truncate` on price span: allows flex item to shrink below content size and truncates with ellipsis if still too long
   - `shrink-0 pl-1` on date span: date never compresses, always stays visible, with 4px left padding as separator
