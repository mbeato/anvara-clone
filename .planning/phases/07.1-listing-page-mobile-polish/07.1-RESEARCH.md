## Summary

This phase is a targeted layout audit of the logged-in listings/browse page (`/listings`) at 375px viewport width. The codebase uses Tailwind CSS v4 (no tailwind.config.js — configured via `@import "tailwindcss"` in `globals.css`) and shadcn/ui components built on Radix UI primitives. No new dependencies are needed for this work.

The core BrowseClient component (`app/(app)/listings/_components/browse-client.tsx`) already has a dual-render pattern for the filter bar (a `sm:hidden` mobile version and a `hidden sm:flex` desktop version), and the CategoryTabRow already uses `overflow-x-auto`. However, a careful source-code audit reveals several concrete issues at 375px that are not visible at desktop widths.

The effective content width at 375px is approximately 311px, not 343px. This is because the app shell `<main>` applies `p-4` (16px sides), and BrowseClient adds another `px-4` on its outer container — yielding 32px total horizontal padding per side (64px total). This double-padding matches the pattern used by all other app pages (`p-6` on page-level containers inside `p-4` main), so it is intentional and should not be changed.

**Primary recommendation:** Fix five concrete layout bugs found by source-code audit. No architectural changes needed — all fixes are targeted Tailwind class adjustments on existing components.

## Architecture Patterns

### Existing Responsive Pattern in This Codebase

The project uses mobile-first Tailwind breakpoints. The `sm:` prefix fires at 640px+. The key breakpoints already established:

- `sm:hidden` — hide element at 640px+ (mobile-only)
- `hidden sm:flex` — hide on mobile, show at 640px+
- `md:` prefix — 768px+
- `lg:` prefix — 1024px+

The dual-render filter bar pattern in `browse-client.tsx` is already in place:

```tsx
{/* Desktop: FilterBar inside the flex row with tabs */}
<div className="hidden sm:flex">
  <FilterBar ... />
</div>

{/* Mobile: FilterBar stacked below tabs */}
<div className="sm:hidden">
  <FilterBar ... />
</div>
```

This pattern is correct and does not need to change.

### Page Content Width at 375px

The app layout adds padding at two levels:
- `app/(app)/layout.tsx`: `<main className="flex flex-1 flex-col gap-4 p-4">` — 16px sides
- `browse-client.tsx`: `<div className="max-w-7xl mx-auto px-4 py-6 space-y-6">` — additional 16px sides

**Effective content width = 375 - 64 = 311px**

All other app pages follow this same double-padding pattern (they use `p-6` inside `p-4` main), so this is correct and should not be changed.

## Concrete Issues Found (Source Audit)

### Issue 1: Loading skeleton overflows horizontally [CONFIRMED]

**File:** `app/(app)/listings/loading.tsx`

**Problem:** Two skeleton rows use plain `flex` with no `overflow-x-auto`:
- Category tab skeletons: `flex gap-2` with 8x `w-20` pills = 8×80px + gaps ≈ 700px — overflows 311px available
- Filter skeletons: `flex gap-3` with 3x `w-40` rectangles = 3×160px + gaps ≈ 500px — overflows 311px available

**Fix:** Add `overflow-x-auto` to both skeleton rows, or reduce the number/width of skeletons to match the actual mobile layout.

Better fix: Mirror the actual mobile layout in the skeleton:
- Tab skeletons: wrap in `overflow-x-auto` (matches CategoryTabRow which already uses `overflow-x-auto`)
- Filter skeletons: show only 2 skeletons (matching the 2 Select dropdowns visible on mobile) stacked vertically, not 3 horizontal

### Issue 2: PropertyCard price row can overflow at 375px [CONFIRMED]

**File:** `app/(app)/listings/_components/property-card.tsx`

**Problem:** The card content area at 375px is ~123px wide (147.5px card - 24px padding). The price row uses `flex items-center justify-between` with no truncation protection:
- Price label: `"From $100,000"` at text-xs ≈ 80-85px
- Date: `"Apr 2025"` at text-xs ≈ 45-50px
- Sum ≈ 130-135px > 123px available

The current `Intl.NumberFormat` uses full currency formatting (`$100,000`) rather than compact notation (`$100K`).

**Fix:** Switch the in-card price display to compact notation (`notation: "compact"`) on mobile. The existing `priceFormatter` variable can be replaced with a compact formatter, since card-level pricing is just a teaser — full pricing is on the detail page. Alternatively: wrap the price span in `min-w-0 truncate` to prevent overflow.

Simplest safe fix: add `truncate` to the price span and `shrink-0` to the date span:
```tsx
<span className="text-xs font-medium truncate">{priceDisplay}</span>
<span className="text-xs text-muted-foreground shrink-0 pl-1">{startDate}</span>
```

### Issue 3: CategoryCarousel negative margin — CONFIRMED NOT AN ISSUE

**File:** `app/(app)/listings/_components/category-carousel.tsx`

**Audit result:** The `CarouselContent` component (`components/ui/carousel.tsx` line 140-143) wraps Embla's `carouselRef` in `<div className="overflow-hidden">`. The `-ml-4` negative margin is applied to the inner flex div, fully contained by `overflow-hidden`. The `category-carousel.tsx` passes `className="-ml-4"` as a prop, but `CarouselContent` already applies `-ml-4` internally via `cn()`. Since the project uses `tailwind-merge`, the duplicate class is deduplicated harmlessly. **No fix needed for the carousel.**

### Issue 4: Property grid is 2-column at 375px — usability check [LIKELY OK]

**File:** `app/(app)/listings/_components/browse-client.tsx`

**Current state:** `grid grid-cols-2 md:grid-cols-3 lg:grid-cols-5 gap-4`

At 375px: each card is ~147.5px wide with a 2:1 aspect image (≈74px tall). This is tight but functional:
- Card name: `text-sm line-clamp-1` — single line, truncates
- Tagline: `text-xs line-clamp-1` — single line, truncates
- Price + date row: 123px available (see Issue 2 above)

The grid itself is correct — 2 columns is the right mobile layout. The issue is only in the price row (see Issue 2). No grid change needed.

### Issue 5: Loading skeleton does not match mobile layout [CONFIRMED, linked to Issue 1]

**File:** `app/(app)/listings/loading.tsx`

The skeleton uses a single `div className="p-6 space-y-6"` (not `px-4 py-6` as BrowseClient uses). This means the skeleton has different padding than the loaded content — causes a layout shift on load. Minor but worth fixing to match `max-w-7xl mx-auto px-4 py-6 space-y-6`.

## Common Pitfalls

### Pitfall 1: Double padding context
**What goes wrong:** Adding `overflow-hidden` to BrowseClient's outer div to prevent horizontal scroll, but the outer div already has `max-w-7xl mx-auto px-4` — `overflow-hidden` here clips the CategoryCarousel's Embla overflow containment.
**How to avoid:** Add `overflow-hidden` only to specific components that need it (like the Carousel), not to the page wrapper.

### Pitfall 2: Skeleton mismatch causing layout shift
**What goes wrong:** The loading skeleton has different padding/structure than the loaded content, causing visible layout shift when data arrives.
**How to avoid:** Mirror the loaded content's outer container classes exactly (`max-w-7xl mx-auto px-4 py-6 space-y-6`).

### Pitfall 3: Tailwind v4 `!` important modifier
**What goes wrong:** Overusing `!mt-2` (already appears in browse-client.tsx) to override `space-y-6` spacing. This can create maintenance confusion.
**How to avoid:** The existing `!mt-2` usages are already in the codebase and intentional — don't add new ones unless needed. Prefer restructuring `space-y-*` containers instead.

### Pitfall 4: `shrink-0` vs `min-w-0` confusion
**What goes wrong:** Adding `shrink-0` to both items in a flex row prevents either from shrinking, causing overflow.
**How to avoid:** In flex rows with text overflow risk: `min-w-0 truncate` on the primary text item (allows shrinking + clips text), `shrink-0` on the secondary item (date/label that should stay full-width).

## Code Examples

### Compact price formatter (already used in filter-bar.tsx)
```typescript
// Source: app/(app)/listings/_components/filter-bar.tsx
const priceFormatter = new Intl.NumberFormat("en-US", {
  style: "currency",
  currency: "USD",
  maximumFractionDigits: 0,
  notation: "compact",
})
// Output: "$25K", "$100K", "$1.5M"
```

### Truncation fix for flex justify-between row with text overflow risk
```tsx
// Fix for PropertyCard price row — prevents overflow at narrow widths
<div className="flex items-center justify-between pt-1">
  <span className="text-xs font-medium min-w-0 truncate">{priceDisplay}</span>
  <span className="text-xs text-muted-foreground shrink-0 pl-1">{startDate}</span>
</div>
```

### Loading skeleton tab row fix (overflow-x-auto to match CategoryTabRow)
```tsx
// Mirror the actual CategoryTabRow's overflow behavior in the skeleton
<div className="flex gap-2 overflow-x-auto scrollbar-hide">
  {Array.from({ length: 6 }).map((_, i) => (
    <Skeleton key={i} className="h-8 w-16 rounded-full shrink-0" />
  ))}
</div>
```

### Loading skeleton filter fix (mobile: 2-column row, not 3 horizontal)
```tsx
// Mobile: 2 select dropdowns stacked, then slider below — mirrors FilterBar mobile layout
<div className="flex flex-col gap-3 sm:hidden">
  <div className="flex gap-3">

  </div>
  <Skeleton className="h-9 w-full rounded-md" />
</div>

    <Skeleton key={i} className="h-10 w-40 rounded-md" />
  ))}
</div>
```
