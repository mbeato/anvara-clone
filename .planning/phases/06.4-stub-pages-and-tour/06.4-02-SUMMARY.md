# Phase 6.4 Plan 02: Tour Modal and Personalize Page Summary

**One-liner:** Multi-step tour Dialog triggered from sidebar onClick replacing broken /tour Link, plus static Personalize settings page with 4 Card sections.

## Accomplishments

- **Tour modal:** 4-step Dialog walkthrough (Welcome, Browse the Marketplace, Track Your Performance, Close Deals Faster) with icons, progress dots, Back/Next/Get Started navigation. Step resets to 0 on close.
- **Sidebar integration:** "Show Me Around" removed from footerItems array, rendered manually as SidebarMenuButton with `onClick={() => setTourOpen(true)}` — no asChild, no href. TourModal placed as sibling to `<Sidebar>` in a fragment.
- **Personalize page:** Static Server Component with 4 settings sections: Notification Preferences (On/Off Badge indicators), Display Preferences (label+value pairs), Marketplace Interests (Badge chips with selected/unselected variants), Account (read-only styled fields).
- **Loading skeleton:** personalize/loading.tsx with heading skeletons + 4 card-shaped skeletons matching dashboard loading.tsx pattern.

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| footerItems array renamed footerLinkItems, Show Me Around extracted | Cleanest approach — keeps array-mapped items uniform (Link/asChild) while handling the special button case explicitly |
| TourModal outside `<Sidebar>` as fragment sibling | Prevents z-index clipping from sidebar's positioning context; Dialog portals to document.body |
| handleClose calls both onOpenChange(false) and setStep(0) | Dialog's onOpenChange fires on X button, overlay click, and Escape — all paths must reset step |
| Personalize non-async, all static | No user preferences model in Prisma schema; prototype-quality static display is correct |
| No Switch component — used Badge On/Off | Research confirmed no Switch in codebase; Badge variant toggle is readable and consistent |
