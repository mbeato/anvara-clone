**Props:**
```typescript
interface FilterBarProps {
  currentFilters: {
    category?: string
    region?: string
    minPrice?: number
    maxPrice?: number
  }
  onFilterChange: (key: string, value: string | undefined) => void
  onPriceChange: (min: number | undefined, max: number | undefined) => void
}
```

**Layout:** Horizontal flex row: `flex flex-wrap items-center gap-3`

**Region dropdown** using ShadCN `<Select>`:
- `<SelectTrigger className="w-40 h-9">` with `<SelectValue placeholder="All Regions" />`
- Options: "All Regions" (value ""), "West", "East", "South", "Midwest"
- `onValueChange` → calls `onFilterChange("region", value || undefined)`
- Controlled via `value={currentFilters.region ?? ""}`

**Event Type dropdown** using ShadCN `<Select>`:
- `<SelectTrigger className="w-40 h-9">` with `<SelectValue placeholder="All Types" />`
- Options: "All Types" (value ""), "Sports", "Music", "Arts", "Food", "Lifestyle"
- `onValueChange` → calls `onFilterChange("category", value || undefined)`
- Controlled via `value={currentFilters.category ?? ""}`
- NOTE: This and the CategoryTabRow both write to the same `category` URL param (per Pitfall 4 in RESEARCH.md). They stay in sync automatically because both read from `currentFilters.category`.

**Price range slider** using ShadCN `<Slider>`:
- Label: `<span className="text-sm text-muted-foreground">Price</span>`
- `<Slider min={0} max={150000} step={5000} value={[currentFilters.minPrice ?? 0, currentFilters.maxPrice ?? 150000]} onValueCommit={handlePriceCommit} className="w-52" />`
- Use `onValueCommit` (not `onValueChange`) to only update URL when user releases the slider thumb — prevents rapid URL updates during drag
- Below the slider, show formatted range: `$0k — $150k` in `text-xs text-muted-foreground`
- `handlePriceCommit` calls `onPriceChange(values[0] || undefined, values[1] < 150000 ? values[1] : undefined)` — omit max if at ceiling to avoid unnecessary filtering

**IMPORTANT:** Use `onValueCommit` on the Slider, not `onValueChange`. The `onValueChange` fires on every pixel drag which would spam router.replace calls. `onValueCommit` fires only when the thumb is released.

If Slider does not support `onValueCommit` (check the installed component), use a local state for the slider value and debounce the URL update with a 300ms setTimeout on `onValueChange`.

**Create `app/(app)/browse/_components/active-filter-chips.tsx`** — "use client" component:

Props:
```typescript
interface ActiveFilterChipsProps {
  currentFilters: {
    category?: string
    region?: string
    minPrice?: number
    maxPrice?: number
  }
  onRemoveFilter: (key: string) => void
  onClearAll: () => void
}
```

1. Compute active chip list from currentFilters — only render chips for non-undefined values
2. Each chip format:
   - Category: capitalize the slug (e.g., "sports" → "Sports")
   - Region: capitalize (e.g., "west" → "West")
   - MinPrice: format as "Min: $XX,XXX" using Intl.NumberFormat
   - MaxPrice: format as "Max: $XX,XXX" — but only if maxPrice < 150000 (otherwise it's the ceiling = no filter)
3. Each chip renders as: `<Badge variant="secondary" className="gap-1 pr-1.5 cursor-pointer hover:bg-destructive/10" onClick={() => onRemoveFilter(key)}>` with label text and an `<X className="size-3" />` icon
4. If 2+ chips exist, render a "Clear all" text button at the end: `<button className="text-xs text-muted-foreground hover:text-foreground underline" onClick={onClearAll}>Clear all</button>`
5. Wrap in `<div className="flex flex-wrap items-center gap-2">` — only render the wrapper if there are active chips (return null otherwise)

Style the chips to echo Anvara's category label styling — use `variant="secondary"` Badge which matches the rounded pill look from PropertyCategories in Phase 3.
