   - Renders as a distinct Card (from shadcn) — NOT a chat bubble. Should be clearly distinguishable.
   - Layout: Card with border-l-4 colored by status:
     - pending: border-l-yellow-500
     - accepted: border-l-green-500
     - declined: border-l-red-500
   - Content:
     - Header row: "Offer Submitted" label + Badge with status (Pending = yellow/outline, Accepted = green, Declined = red)

     - Tier: "Tier: {tierName}" in muted text

     - Timestamp: small muted text at bottom right

     - Optional message/note: Input text field, placeholder "Add a note (optional)"
   - Buttons: "Send Offer" (primary, disabled when submitting) + "Cancel" (ghost/outline)
   - Form validation: amount required and > 0, tier selection required
   - On submit: call onSubmit with (amount, selectedPackageName, note)

   **State additions:**
   - `showOfferForm`: boolean, default false
   - `isSubmittingOffer`: boolean, default false

   **"Make an Offer" button:**
   - Add a button in the conversation header area (next to property name) or above the message input
   - Label: "Make an Offer" with DollarSign icon (from lucide-react)
   - onClick: `setShowOfferForm(true)`
   - Disabled when isTyping or isSubmittingOffer

   **Render InlineOfferForm:**
   - When showOfferForm is true, render InlineOfferForm ABOVE the message input (below messages, above input — within the bottom area)
   - Pass property.packages as packages prop
   - onCancel: `setShowOfferForm(false)`
   - isSubmitting: isSubmittingOffer
   - onSubmit handler (handleOfferSubmit):
     1. setIsSubmittingOffer(true)
     2. Call createOfferInThread(thread.id, amount, packageName, note)
     3. If success:
        a. Add the [OFFER] message to local messages state (as advertiser bubble — but will render as OfferCard)
        b. setShowOfferForm(false)
        c. setIsTyping(true) — AI will respond to the offer
        d. Trigger AI response: fetch("/api/chat", { method: "POST", body: JSON.stringify({ threadId: thread.id }) })
        e. Add AI response to messages
        f. setIsTyping(false)
     4. If failure: show error
     5. setIsSubmittingOffer(false)

   - When mapping messages, check if message content starts with "[OFFER]"

   - Pass the latest matching offer's status from thread.offers array
   - Parse amount from "[OFFER] $X,XXX — TierName" format:
     - Extract amount: parseInt(content.match(/\$([0-9,]+)/)?.[1]?.replace(/,/g, ""))
     - Extract tier: content.match(/— (.+?)(?:\. Note:|$)/)?.[1]
     - Extract note: content.match(/Note: (.+)$/)?.[1]
