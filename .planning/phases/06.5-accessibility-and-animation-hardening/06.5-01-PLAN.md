  <files>

  </files>
  <action>
    1. In `globals.css`, add a `@media (prefers-reduced-motion: reduce)` block AFTER the existing `@layer base` block:
       ```css
       @media (prefers-reduced-motion: reduce) {
         /* Disable CSS keyframe animations for users who prefer reduced motion */
         [class*="animate-"] {
           animation-duration: 0.01ms !important;
           animation-iteration-count: 1 !important;
         }

         /* Disable CSS transitions globally for reduced motion */
         *,
         *::before,
         *::after {
           transition-duration: 0.01ms !important;
         }
       }
       ```
       - This covers: marquee (animate-[marquee_30s...]), animate-pulse (cursor blinks), any Tailwind animate- class
       - Using `[class*="animate-"]` targets Tailwind animation utilities specifically
       - The global `*` transition rule also catches inline CSS transitions (used in anvara-intelligence and performance-reporting)
       - `0.01ms` (not 0ms) avoids potential issues with transitionend events that some components may listen for

    2. In `what-is-anvara.tsx`, replace all `duration-700` classes with `duration-300`:
       - Line with `transition-all duration-700 ease-out` on the token wrapper div — change to `duration-300`
