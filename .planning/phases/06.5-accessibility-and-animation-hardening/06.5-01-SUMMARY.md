# Phase 6.5 Plan 01: LazyMotion Bundle Optimization and Accessibility Summary

**One-liner:** LazyMotion domAnimation wrapper cuts Motion bundle ~34kb to ~4.6kb; OS prefers-reduced-motion respected via MotionConfig + CSS media query; all animation durations under 400ms.

## Objective

Reduce Motion JS bundle size, respect user accessibility preferences, and tighten animation timing throughout the landing page. No animation exceeds 400ms after this plan.

## Tasks Completed

### Task 1: Create LandingProviders and migrate Motion components

Created `app/_components/landing/landing-providers.tsx` — a `"use client"` wrapper that provides:
- `LazyMotion features={domAnimation} strict` — loads only the ~4.6kb domAnimation feature set instead of the full ~34kb motion bundle
- `MotionConfig reducedMotion="user"` — tells Motion to read the OS `prefers-reduced-motion` setting and disable animations accordingly

Migrated motion components:
- `section-reveal.tsx`: `import { motion }` → `import * as m from "motion/react-m"`, `motion.div` → `m.div`, duration 0.55 → 0.4
- `hero-section.tsx`: Split import so `AnimatePresence` stays in `motion/react`, `m.*` namespace from `motion/react-m`; both `motion.div` instances replaced with `m.div`; video crossfade `duration-1000` → `duration-300`
- `page.tsx`: Entire `<main>` wrapped in `<LandingProviders>` — preserves server component rendering for the page itself

### Task 2: Add CSS prefers-reduced-motion and tighten what-is-anvara durations

Added `@media (prefers-reduced-motion: reduce)` block to `globals.css`:
- `[class*="animate-"]` selector disables Tailwind keyframe utilities (marquee, animate-pulse cursor blink)
- Global `*` transition override catches inline CSS transition styles

Replaced all `duration-700` with `duration-300` in `what-is-anvara.tsx`:
- Token wrapper div (transform/scale/opacity transitions)
- Glass card (box-shadow transition)
- Icon mask div (backdrop-filter transitions)

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| `AnimatePresence` stays in `motion/react` | AnimatePresence orchestrates exit animations but is not a motion component — no code-split benefit, and importing from `motion/react-m` would break strict mode |
| `strict` prop on LazyMotion | Catches any remaining `motion.div` usages at dev-time with a descriptive error |
| `0.01ms` not `0ms` for reduced-motion | Some component libraries listen for `transitionend` events; `0ms` can skip the event entirely |
| LandingProviders wraps only landing page | Other app pages (dashboard, listings) use separate client boundaries and don't need Motion context |
