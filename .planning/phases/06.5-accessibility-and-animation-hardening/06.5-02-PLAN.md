<task type="auto">
  <name>Task 2: Add useReducedMotion to JS-animated sections and tighten durations</name>
  <files>
    app/_components/landing/anvara-intelligence.tsx
    app/_components/landing/performance-reporting.tsx
  </files>
  <action>
    These two components use IntersectionObserver + inline CSS transitions (not Motion components), so MotionConfig's reducedMotion="user" does NOT cover them. They need the useReducedMotion hook from motion/react.

    NOTE: The CSS `prefers-reduced-motion` media query added in Plan 01's globals.css ALSO handles these inline CSS transitions via the global `* { transition-duration: 0.01ms !important }` rule. The useReducedMotion hook here is a defense-in-depth approach that also skips the IntersectionObserver-driven state change entirely, showing the final visible state immediately without any JS-driven transition.

    1. In `anvara-intelligence.tsx` — AnimatedFlow component:
       - Import `useReducedMotion` from `motion/react`
       - Call `const prefersReduced = useReducedMotion()` inside AnimatedFlow
       - If `prefersReduced` is true, set `visible` to true immediately (skip IntersectionObserver):
         ```tsx
         useEffect(() => {
           if (prefersReduced) { setVisible(true); return; }
           // ... existing IO logic
         }, [prefersReduced]);
         ```
       - Tighten durations:
         - `stroke-dashoffset 0.6s` → `stroke-dashoffset 0.4s` (FlowCenter SVG dash lines, line ~159)
         - `transitionDuration: "0.6s"` → `transitionDuration: "0.4s"` (PromptFormCard slide-in, line ~230 area)
         - Any other `0.6s` transition durations in FlowCenter → `0.4s`

    2. In `performance-reporting.tsx` — MetricsVisualization component:
       - Import `useReducedMotion` from `motion/react`
       - Call `const prefersReduced = useReducedMotion()` inside MetricsVisualization
       - If `prefersReduced` is true, set `visible` to true immediately:
         ```tsx
         useEffect(() => {
           if (prefersReduced) { setVisible(true); return; }
           // ... existing IO logic
         }, [prefersReduced]);
         ```
       - Change `const dur = "1.2s"` to `const dur = "0.5s"` (SVG curve draw animation)

    IMPORTANT: `useReducedMotion` returns `boolean | null` — null means "not yet determined" (SSR). Treat null as false (animate normally), only skip on explicit true:
    ```tsx
    if (prefersReduced === true) { setVisible(true); return; }
    ```
  </action>
  <verify>
    Run `npx tsc --noEmit` — no TypeScript errors. Grep for "useReducedMotion" in both files — should return matches. Grep for `"1.2s"` in performance-reporting.tsx — should return 0 results. Grep for `"0.6s"` in anvara-intelligence.tsx — should return 0 results. Run `npm run build` to verify clean build.
  </verify>
  <done>
    Both anvara-intelligence.tsx and performance-reporting.tsx use useReducedMotion to skip animations for reduced-motion users. Anvara Intelligence transitions tightened from 0.6s to 0.4s. Performance Reporting SVG draw tightened from 1.2s to 0.5s. Build passes cleanly.
  </done>
