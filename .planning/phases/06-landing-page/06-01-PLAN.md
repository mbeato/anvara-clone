2. Create `app/_components/landing/section-reveal.tsx` — a "use client" wrapper that uses motion's `whileInView` for scroll-triggered fade-up reveals:

```tsx
"use client";
import { motion } from "motion/react";

interface SectionRevealProps {
  children: React.ReactNode;
  delay?: number;
  className?: string;
}

export function SectionReveal({ children, delay = 0, className }: SectionRevealProps) {
  return (
    <motion.div
      initial={{ opacity: 0, y: 32 }}
      whileInView={{ opacity: 1, y: 0 }}
      viewport={{ once: true, amount: 0.15 }}
      transition={{ duration: 0.55, ease: [0.25, 0.1, 0.25, 1], delay }}
      className={className}
    >
      {children}
    </motion.div>
  );
}
```

Key decisions:
- `viewport.once: true` — animates on first entry only (premium feel, not distracting on scroll-back)
- `viewport.amount: 0.15` — triggers when 15% visible (catches large sections early)
- Custom cubic bezier easing for smooth deceleration
- `delay` prop allows staggering multiple elements within a section

  <action>
Add custom Tailwind v4 animation keyframes inside the existing `@theme inline` block in `app/globals.css`. These provide the CSS-only marquee for the Final CTA section and a fade-up utility.

Inside the `@theme inline { ... }` block, add these animation custom properties and keyframes:

```css
  --animate-marquee: marquee 40s linear infinite;
  --animate-marquee-slow: marquee 60s linear infinite;
```

After the closing `}` of `@theme inline`, add:

```css
@keyframes marquee {
  from { transform: translateX(0); }
  to { transform: translateX(-50%); }
}
```

The marquee works by duplicating content and translating exactly -50% — when the first set scrolls out, the duplicate is seamlessly in position. The 40s duration provides a smooth, non-rushed feel for 12 property cards. The slow variant (60s) is available if needed.

Do NOT add the motion-based fade-up here — that's handled by SectionReveal component with motion's whileInView (more reliable than CSS-only intersection observer).
