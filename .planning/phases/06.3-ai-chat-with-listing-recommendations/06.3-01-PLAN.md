<task type="auto">
  <name>Task 1: Create /api/ai-chat POST route with property catalog context</name>
  <files>app/api/ai-chat/route.ts</files>
  <action>
Create `app/api/ai-chat/route.ts` that mirrors the pattern from `app/api/chat/route.ts` but serves AI chat recommendations instead of thread-based messaging.

**Implementation:**

1. Export `maxDuration = 60` (same as existing `/api/chat` route — o3-mini can take 10-30s).

2. Define a `ChatMessage` type: `{ role: "user" | "assistant"; content: string }`.

3. Create `buildSystemPrompt(properties)` that:
   - Takes the return type of `getAllProperties()`
   - Maps each property to a compact JSON object with: `slug`, `name`, `category`, `subcategory`, `location`, `tags` (joined as comma string), `priceFrom`, `audienceAgeRange`, `description` (truncated to 200 chars)
   - Returns a system prompt string that:
     - Identifies the AI as "Anvara AI, a sponsorship marketplace assistant"
     - Includes the property catalog as a JSON array
     - Instructs the AI to respond as JSON: `{ "text": "your response", "recommendedSlugs": ["slug1", "slug2"] }`
     - Instructs to recommend UP TO 3 relevant listings
     - Instructs to return empty `recommendedSlugs` array if no listings match
     - Instructs to keep responses concise and actionable (2-3 sentences)
     - Instructs to reference specific property names and categories

4. Export `POST` handler that:
   - Parses body as `{ messages: ChatMessage[] }`
   - Calls `getAllProperties()` to get the full catalog
   - Calls `generateText` with `openai("o3-mini")`, the system prompt, and the messages array
   - Wraps `JSON.parse(text)` in try/catch — fallback to `{ text: rawText, recommendedSlugs: [] }` if AI returns non-JSON
   - Validates recommended slugs: build a `Set` of known slugs from properties, filter the AI's slugs against it, slice to max 3
   - Returns `Response.json({ text: parsed.text, recommendedSlugs: validatedSlugs })`
   - Catches errors and returns 500 with `{ error: "Failed to generate response" }`

**DO NOT:** Use `@ai-sdk/react` (not installed). DO NOT persist anything to the database. DO NOT import prisma directly — use `getAllProperties` from `lib/data`.
  </action>
  <verify>Run `npx tsc --noEmit` and confirm no TypeScript errors in the new file. Verify the file exports `POST` and `maxDuration`.</verify>
  <done>The /api/ai-chat route accepts POST with messages array, calls o3-mini with property catalog context, returns { text, recommendedSlugs } with validated slugs, and handles JSON parse failures gracefully.</done>
</task>
