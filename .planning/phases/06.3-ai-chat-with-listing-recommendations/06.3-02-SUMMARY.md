## Decisions Made
- UserMessage and AIMessage have no `"use client"` — pure presentational, imported by client parent (AIChatClient). This avoids unnecessary client boundary boundaries on leaf nodes.
- `recommendedSlugs` stripped before API POST — only `{ role, content }` sent per message. Prior recommendations are display-only metadata and should not pollute conversation history sent to the AI.
- `void` operator on floating promises in onClick/onKeyDown — satisfies TypeScript strict mode without making event handlers async.
