# Phase 5.1 Plan 01: Enforce Coding Standards — Cleanup Pass 1 Summary

Removed seven React anti-patterns across six files: one spurious useEffect skeleton delay, one dead useEffect import, three unnecessary "use client" directives on pure Server Components, one redundant eslint-disable suppression, and one bare Suspense wrapper wrapping already-resolved data.

## Accomplishments

- **browse-client.tsx:** Removed artificial 300ms skeleton loading delay (`isLoading` state + `useEffect` setTimeout). The route-level `loading.tsx` already handles navigation skeletons; the client-side delay was pure overhead. Removed `PropertyCardSkeleton` import and the conditional skeleton render branch. Refactored `removeFilter` to build URL params directly (same pattern as `handlePriceChange`) — eliminates the `eslint-disable-next-line react-hooks/exhaustive-deps` suppression entirely.
- **az-execution.tsx:** Removed `useEffect` from the React import line. The symbol was imported but never referenced in the component body.
- **for-brands-rightsholders.tsx:** Removed `"use client"` directive. Component has no hooks, no state, no browser APIs. `SectionReveal` and `Link` carry their own client boundaries.
- **faq-section.tsx:** Removed `"use client"` directive. ShadCN's `Accordion` component has an internal `"use client"` boundary. The `FaqSection` parent renders static JSX + Accordion children only.
- **recommendations-strip.tsx:** Removed `"use client"` directive. Component maps over a `properties` prop and renders `PropertyCard` — pure Server Component render.
- **messages/page.tsx:** Removed bare `<Suspense>` wrapper around `<MessagingClient>`. The page already `await`s `getThreads()` before render; Suspense with no fallback served no purpose. Removed the now-unused `Suspense` import from React.

| Replace skeleton useEffect with nothing | Next.js App Router loading.tsx is the correct mechanism for navigation loading states; client setTimeout creates a fake delay that harms actual UX |
| Refactor removeFilter to direct param construction | Matches the established handlePriceChange pattern; eliminates eslint-disable suppression which masked a real dependency tracking issue |
