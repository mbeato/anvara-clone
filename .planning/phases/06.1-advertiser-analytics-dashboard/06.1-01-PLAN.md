Create `lib/data/analytics.ts` — a pure derivation module that takes pre-fetched data as arguments (no Prisma imports, no async calls). This file derives all metrics from seeded property and thread data.

**Type definitions:** Use `Awaited<ReturnType<typeof getAllProperties>>` and `Awaited<ReturnType<typeof getThreads>>` pattern to derive types from the existing data helpers. Import these as `import type`.

**Export a single `deriveAnalytics(properties, threads)` function** that returns an object with:

1. **`kpis`** — array of 4 KPI objects:
   - "Total Impressions": sum of all `audienceTotalReach`, formatted (e.g., "42.3M"), trend +14.2%
   - "Active Campaigns": `threads.length`, trend +8.1%
   - "Conversion Rate": `acceptedOffers / totalOffers * 100` (fallback to 12.4% if no accepted offers), trend +2.3%
   - "Deal Pipeline": sum of all `offer.amount` formatted as "$583K", trend +22.7%
   - Each KPI has: `{ label: string, value: string, trend: number, iconName: "ChartArea" | "TrendingUp" | "Briefcase" | "MessageSquare" }`
   - Note: use iconName strings (not icon components) since this data crosses server/client boundary

2. **`impressionsTrend`** — array of 30 objects `{ date: string, impressions: number }`:
   - Generate deterministically from `totalReach` (NO `Math.random()`)
   - Use seeded variation: `(base * (i + 1) * 7919) % 1000` for variance
   - Slight upward trend: multiply by `(1 + i * 0.01)`
   - Format dates as "Feb 1", "Feb 2" etc using `toLocaleDateString("en-US", { month: "short", day: "numeric" })`
   - Use a fixed anchor date (e.g., `new Date(2026, 1, 22)`) instead of `Date.now()` to ensure determinism across renders

3. **`categoryBreakdown`** — array of objects `{ category: string, properties: number, totalReach: number, avgPrice: number }`:
   - Group properties by `category`, capitalize first letter
   - Calculate count, sum of audienceTotalReach, average of priceFrom per category

4. **`recentActivity`** — array of objects `{ propertyName: string, type: "offer" | "message", detail: string, date: string }`:
   - Derive from threads: for each thread, create entries for its latest message and any offers
   - Format dates as relative-ish strings (use `createdAt.toLocaleDateString()`)
   - Sort by date descending, take top 8

**Helper functions** (also exported for reuse):
- `generateImpressionsTrend(totalReach: number, dayCount?: number)`
- `deriveKpis(properties, threads)`
- `deriveCategoryBreakdown(properties)`
- `deriveRecentActivity(threads)`
- `formatReach(n: number): string` — "42.3M", "583K", "1,234" formatting

All values must be serializable (no Date objects in return types — convert to strings).
