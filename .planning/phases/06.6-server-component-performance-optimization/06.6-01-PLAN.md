<objective>
Remove unnecessary `"use client"` directives from two display-only components that have no hooks, no state, and no browser API usage. These components are pure renderers of serializable props and should be Server Components, reducing client bundle size and correctly signaling to Next.js that they require no client JavaScript.

Note: `empty-state.tsx` and `active-filter-chips.tsx` were excluded because they accept function props (`onClearFilters`, `onRemoveFilter`, `onClearAll`) from the Client Component `browse-client.tsx`. Functions are not serializable across the Server/Client boundary, so removing `"use client"` from those files would cause a build failure.

Purpose: Enforce the Server Component performance pattern — `"use client"` only on components that genuinely use hooks or browser APIs, never on display-only components.
Output: Two files with `"use client"` removed, clean build confirmation.
</objective>
